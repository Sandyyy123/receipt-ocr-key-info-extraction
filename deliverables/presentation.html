Receipt OCR and Key Information Extraction on SROIE

Motivation

Accounts-payable (AP) automation is the highest-volume document-intelligence task in the DACH Mittelstand. A typical mid-size German firm processes 5,000-50,000 invoices per year, and four header fields (vendor, date, address, total) carry most of the downstream business value.

Business framing. At 4-6 minutes of manual data entry per invoice, every percentage point of micro-F1 lifted above the rule-based baseline saves approximately 30-50 hours per year on a 50,000-invoice queue. A 5-point lift from LayoutLMv3 over Tesseract+regex pays back the GPU and engineering investment within months.

The same architecture transfers directly to German Rechnungen and Lieferscheine after a small fine-tune on a domain corpus, which is the planned v1.0 extension.

Dataset

SROIE v2 (ICDAR 2019 Robust Reading Challenge on Scanned Receipt OCR and Information Extraction). Public Kaggle mirror at urbikn/sroie-datasetv2, ~200 MB.

  • 1,000 receipts from Singapore and Malaysia merchants.
  • Per-receipt per-word bounding boxes (box/*.txt).
  • Per-receipt JSON ground truth for four fields: company, date, address, total (entities/*.txt).
  • Train / test split provided by ICDAR. We carve a 10% validation set from train (seed 42) for early stopping.

Class imbalance under BIO tagging is severe: O tokens dominate (>85%) on most receipts, which motivates seqeval entity-level F1 over token accuracy.

Methods

Two pipelines, one evaluation harness

Both pipelines share the same train/val/test split, the same BIO label generation, and the same per-field exact-match F1 evaluator. Predictions and metrics are written as JSON to deliverables/ so that the manuscript build script can read them programmatically and inject numbers without hand-editing.

Baseline pipeline

  1. Image preprocessing: deskew via minAreaRect, adaptive Gaussian thresholding.
  2. Tesseract OCR (--oem 3 --psm 6, language eng).
  3. Field-specific extractors (regex + spatial heuristics).

Advanced pipeline

  1. Read pre-computed word boxes; assign BIO labels by greedy substring alignment to ground truth.
  2. LayoutLMv3-base fine-tune via HuggingFace Trainer, 8 epochs, batch 4, lr 5e-5, fp16.
  3. Best checkpoint by val seqeval F1 reloaded for test evaluation.
Unsloth alternative. For shared-GPU scenarios, Unsloth provides a drop-in QLoRA path that reduces VRAM by ~60% with comparable F1. The implementation documents the substitution as a single import change.

Discussion

Limitations

  • SROIE receipts are English-language Asian retail formats; transfer to German Rechnungen requires re-fine-tune on a German corpus and switching Tesseract to deu+eng.
  • SROIE provides only four target fields; production AP also extracts VAT IDs, line items, and bank details.
  • BIO label generation by greedy substring alignment introduces label noise when the ground-truth string differs from the printed string by a typographic correction.

Future work

  • OCR-free comparison via Donut.
  • Full QLoRA fine-tune via Unsloth.
  • Transfer to a German Rechnung corpus.

Conclusion

This Initial implementation delivers two fully-specified, runnable pipelines for SROIE Task 3 under a shared evaluation harness. Once the test-split numbers are populated, per-field F1 deltas, CPU-versus-GPU latency, and training cost are all accessible from the same JSON files, and the cost-quality trade-off can be presented to a non-technical sponsor in a single dashboard view.
